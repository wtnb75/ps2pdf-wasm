# Ghostscript wasm ブラウザ版PoC Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild Ghostscript's `gs` as wasm with a browser-compatible emscripten configuration (MEMFS + `--preload-file` + `MODULARIZE`), and prove that a static HTML page can load it in a browser and convert a PostScript file to PDF.

**Architecture:** Reuse the same `ghostscript-10.07.1/` source tree and `emconfigure`/`emmake` build already verified for the Node PoC (`poc/`), but re-link `bin/gs` with browser-targeted flags (`-sENVIRONMENT=web -sMODULARIZE=1 -sEXPORT_NAME=createGSModule -sEXPORTED_RUNTIME_METHODS=callMain,FS --preload-file Resource@/Resource --preload-file lib@/lib`, dropping `-sNODERAWFS=1`). Since the configure step and compiled `.o` files are identical to the Node build, only the link step re-runs (after removing the previous `bin/gs`/`bin/gs.wasm` link outputs to force a relink). The resulting `gs.js`/`gs.wasm`/`gs.data` are copied into a new `poc-browser/` directory alongside a minimal `index.html` that runs the conversion on page load and a `test.ps` (copied from `poc/test.ps`). Verification is done by serving `poc-browser/` with `python3 -m http.server` and checking the page output.

**Tech Stack:** emscripten 3.1.69 (emcc/emconfigure/emmake, already installed), GNU make, Python 3 (`http.server`), the bundled `ghostscript-10.07.1` autotools build (already configured for wasm from the Node PoC).

## Verified facts (from Node PoC, do not re-derive)

- `ghostscript-10.07.1/` already has a `Makefile` generated by `emconfigure ./configure --host=wasm32-unknown-emscripten --with-drivers=pdfwrite --disable-dynamic --disable-contrib --disable-cups --disable-fontconfig --disable-gtk --without-x --without-libtiff --without-pdftoraster --without-ijs --without-tesseract`, with `CC=/usr/share/emscripten/emcc`, `CCAUX=gcc`, `CXX=/usr/share/emscripten/em++`. This configuration is target-architecture-independent (wasm32), so it does NOT need to be re-run for the browser build — the same `Makefile`/`obj/*.o` are reused.
- `emmake make gs` already succeeded once (for the Node build) and produced `bin/gs` (JS launcher, extensionless) + `bin/gs.wasm`, linked via `GS_LDFLAGS=$(LDFLAGS)=$(AC_LDFLAGS) $(XLDFLAGS)`. Only `XLDFLAGS` needs to change for the browser build; the compiled `.o` files are unaffected and will not be recompiled.
- Because `bin/gs` and `bin/gs.wasm` already exist (from the Node build) and are newer than the `.o` files, `make` will consider the link target up to date and will NOT re-link automatically even if `XLDFLAGS` changes. The existing `bin/gs`/`bin/gs.wasm` (and any `bin/gs.data`) must be removed before re-running `emmake make gs` to force a relink with the new flags.
- `ghostscript-10.07.1/Resource/` and `ghostscript-10.07.1/lib/` are real directories containing Ghostscript's init scripts and resources (used as `GS_LIB` in the Node PoC).

### Task 2: Build the browser-targeted `gs.js`/`gs.wasm`/`gs.data`

**Files:**
- Create: `poc-browser/build.sh`
- Modify: `ghostscript-10.07.1/bin/gs*` (build outputs, not committed)

- [ ] **Step 1: Create the build script**

Create `poc-browser/build.sh`:

```bash
#!/bin/bash
set -euo pipefail

ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
GS_SRC="$ROOT/ghostscript-10.07.1"
POC="$ROOT/poc-browser"

cd "$GS_SRC"

if [ ! -f Makefile ] || ! grep -q "^CC=/usr/share/emscripten/emcc" Makefile; then
  emconfigure ./configure --host=wasm32-unknown-emscripten \
    --with-drivers=pdfwrite \
    --disable-dynamic \
    --disable-contrib \
    --disable-cups \
    --disable-fontconfig \
    --disable-gtk \
    --without-x \
    --without-libtiff \
    --without-pdftoraster \
    --without-ijs \
    --without-tesseract
fi

# Remove previous link outputs (e.g. from the Node PoC build) so that the
# link step always re-runs with the browser-targeted XLDFLAGS below.
rm -f bin/gs bin/gs.js bin/gs.wasm bin/gs.data

emmake make gs -j4 \
  XLDFLAGS="-sALLOW_MEMORY_GROWTH=1 -sINITIAL_MEMORY=67108864 -sEXIT_RUNTIME=0 -sENVIRONMENT=web -sMODULARIZE=1 -sEXPORT_NAME=createGSModule -sEXPORTED_RUNTIME_METHODS=callMain,FS --preload-file Resource@/Resource --preload-file lib@/lib"

cp bin/gs "$POC/gs.js"
cp bin/gs.wasm bin/gs.data "$POC/"

echo "Build artifacts copied to $POC/"
```

- [ ] **Step 1: Create the verification page**

Create `poc-browser/index.html`:

```html
<!DOCTYPE html>
<html>
<head><meta charset="utf-8"><title>Ghostscript wasm Browser PoC</title></head>
<body>
  <h1>Ghostscript wasm Browser PoC</h1>
  <pre id="log"></pre>
  <a id="download" style="display:none">out.pdf をダウンロード</a>
  <script src="gs.js"></script>
  <script>
    const log = (msg) => { document.getElementById('log').textContent += msg + '\n'; };

    createGSModule().then(async (Module) => {
      const psText = await (await fetch('test.ps')).text();
      Module.FS.writeFile('/test.ps', psText);

      Module.callMain([
        '-sDEVICE=pdfwrite',
        '-dNOPAUSE',
        '-dBATCH',
        '-I/lib', '-I/Resource/Init',
        '-sOutputFile=/out.pdf',
        '/test.ps',
      ]);

      const pdfBytes = Module.FS.readFile('/out.pdf');
      const header = new TextDecoder().decode(pdfBytes.slice(0, 4));
      if (header !== '%PDF') {
        throw new Error(`Expected %PDF, got: ${header}`);
      }
      log('PoC OK: out.pdf starts with %PDF');

      const blob = new Blob([pdfBytes], { type: 'application/pdf' });
      const a = document.getElementById('download');
      a.href = URL.createObjectURL(blob);
      a.download = 'out.pdf';
      a.style.display = 'inline';
    }).catch((e) => {
      log('PoC FAILED: ' + e.message);
      throw e;
    });
  </script>
</body>
</html>
```
